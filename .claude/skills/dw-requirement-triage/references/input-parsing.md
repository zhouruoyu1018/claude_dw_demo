# 输入解析规则

## Step 0: 交互图解析（如有）

当用户提供了交互图（截图、原型图、设计稿）时，优先执行本步骤。通过 Read 工具读取图片文件，进行视觉解析。

**解析目标**：从交互图中提取以下数仓相关元素：

| UI 元素 | 映射目标 | 提取示例 |
|---------|---------|---------|
| **筛选器/筛选条件** | 维度 | 日期选择器 → `stat_date`；渠道下拉框 → `channel` |
| **表格列头** | 指标 + 维度 | "放款金额(万元)" → 指标；"产品名称" → 维度 |
| **图表 X/Y 轴标签** | 时间粒度 + 指标 | X轴"2024-01~2024-12" → 月粒度；Y轴"逾期率%" → 指标 |
| **Tab/切换器** | 多粒度/多维度需求 | "按日｜按月｜按年" → 需支持多时间粒度 |
| **导出按钮** | 导出需求 | "导出 Excel" → 标记数据导出需求 |
| **下钻链接/跳转箭头** | 层级关系 | "点击查看明细 →" → 需要 DA 层明细表 |
| **数字卡片/KPI 区域** | 核心指标 | "今日放款 ¥1,234万" → 指标 `loan_amount` |
| **图表类型** | 分析类型 | 折线图 → 趋势分析；饼图 → 占比分析；柱状图 → 对比分析 |

**解析流程**:
```
读取交互图
    ↓
识别页面类型（报表页/大屏/列表页/详情页/表单页）
    ↓
├─ 报表页/大屏 → 高概率数仓需求，提取全部 UI 元素
├─ 列表页 → 关注筛选器和统计汇总区域
├─ 详情页 → 关注是否有统计指标区域
└─ 表单页 → 通常非数仓需求，但注意下拉选项是否来自维度表
    ↓
输出: 交互图解析结果（维度列表、指标列表、时间粒度、图表类型、下钻关系）
```

**多张交互图处理**: 如果用户提供多张交互图，逐张解析并标注页面编号，在后续需求中关联来源图。

## Step 0.1: Excel 导出模板解析（如有）

当用户提供了 Excel 导出模板（.xlsx/.xls 文件或 Excel 截图）时，执行本步骤。Excel 模板比交互图和文本更精确，因为字段名、数据格式、列顺序都是明确定义的。

**解析目标**：从 Excel 模板中提取以下数仓相关元素：

| Excel 元素 | 映射目标 | 提取示例 |
|-----------|---------|---------|
| **列头（表头行）** | 字段名 → 维度或指标 | "产品名称" → 维度；"放款金额(万元)" → 指标 |
| **数据格式** | 字段类型与精度 | 百分比(2位) → `DECIMAL(10,4)`；日期 → `DATE` |
| **示例数据** | 口径理解线索 | "2.35%" 在"逾期率"列 → 辅助判断是金额口径还是笔数口径 |
| **合计行/小计行** | 聚合维度与方式 | 按渠道小计 → `GROUP BY channel`；总计行 → 全局 SUM |
| **多 Sheet** | 多维度视角或多报表 | Sheet1"按日汇总"、Sheet2"按月汇总" → 多粒度需求 |
| **冻结列** | 维度列识别 | 左侧冻结的列通常是维度（如日期、渠道、产品） |
| **合并单元格** | 层级维度关系 | 一级分类合并多行 → 父子维度层级 |
| **条件格式/颜色标记** | 业务规则线索 | 红色标记逾期率>5% → 阈值告警需求 |
| **公式** | 衍生指标计算逻辑 | `=C2/B2` 在"逾期率"列 → `overdue_amount / loan_amount` |

**解析流程**:
```
读取 Excel 模板（文件或截图）
    ↓
识别模板类型
    ↓
├─ 明细导出模板 → 每列对应一个输出字段，确定导出粒度
├─ 汇总报表模板 → 识别维度列 + 指标列 + 聚合方式
├─ 透视表模板 → 行维度 + 列维度 + 值指标
└─ 混合模板 → 多 Sheet 分别识别
    ↓
提取: 字段清单（名称、类型、格式）、聚合方式、维度层级、数据粒度
    ↓
反推: 需要哪些源表字段、需要什么聚合逻辑、数据量级预估
```

**Excel 模板的独特价值**（文本和交互图通常无法提供的）:
- **精确字段名**: 列头就是业务方期望的输出字段名，可直接映射为表字段
- **数据格式要求**: 百分比、金额精度、日期格式 → 直接确定字段类型
- **计算公式**: 公式暴露了指标间的计算关系 → 直接转化为 ETL 逻辑
- **数据量线索**: 示例数据行数 + 日期跨度 → 预估全量数据规模
- **排序需求**: 默认排序列 → 影响 Doris 表的排序键设计

**多 Sheet 处理**: 逐个 Sheet 解析，每个 Sheet 可能对应不同的报表需求或同一报表的不同粒度视角。
