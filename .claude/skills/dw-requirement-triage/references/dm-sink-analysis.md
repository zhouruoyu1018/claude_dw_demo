# Step 8.4: 单表 dm 下沉分析

即使是单表需求，当目标层为 DA 时，也应分析其中的指标是否具有通用复用价值，若有则建议先落到 DM 层再被 DA 引用，避免通用指标被锁死在特定报表中。

**触发条件**: Step 8.1 判定为单表模式 **且** 需求清单中建议分层为 DA（不区分大小写，与 Step 7 输出模板一致）。DM 层目标不触发（DM 本身就是中间层）。

## 覆盖场景

| # | 场景 | 示例 |
|---|------|------|
| S6 | da 含通用指标 | da 报表算了"放款金额、放款笔数、逾期率"，这些是通用指标，应先落 dm |
| S7 | da 与现有 dm 部分重叠 | 新 da 需要 5 个指标，其中 3 个已在某 dm 宽表中，另 2 个应扩列到该 dm |
| S8 | 指标计算链 | 指标 A 依赖指标 B，B 依赖 C，链长 >= 3，中间结果应落 dm 供复用 |

## 执行流程

```
Step 8.1 → 单表模式 + 目标层=DA
    ↓
Step 8.4: 单表 dm 下沉分析
    ↓
├─ 判定需要下沉 → 升级为多表模式（dm + da）
└─ 判定不需要 → 保持单表，正常进入 Phase 2
```

## 8.4.1 指标通用度评估

对目标表中的每个指标，判断其"通用度":

```
通用信号（满足越多越通用）:
  [+3] 指标名不含报表/场景限定词（如"日报专用"、"导出用"）
  [+3] 指标属于标准业务域指标（放款金额/笔数、逾期率/金额、还款金额等）
  [+2] 指标已在 indicator_registry 中登记（调用 search_existing_indicators）
  [+2] 指标计算逻辑复杂（窗口函数/多表JOIN），复用价值高
  [+1] 指标可被简单聚合到其他粒度（SUM/COUNT/AVG）

专用信号（满足越多越专用）:
  [-3] 指标含报表特定计算（如排名、占比、环比）
  [-2] 指标依赖报表维度组合（如"产品x渠道交叉占比"）
  [-1] 指标仅在当前报表的交互图/Excel模板中出现

通用度 = sum(通用信号) - sum(专用信号)
```

## 8.4.2 现有 dm 表搜索

对通用度 >= 3 的指标，搜索是否有同主题同粒度的现有 dm 表:

```
search_existing_indicators(指标名) → 找到已有表?
search_table(主题关键词) → 找到同粒度 dm 表?
MEMORY.md 表清单 → 近期新建的 dm 表?
```

**MCP 不可用时降级**: 跳过自动搜索，将"现有 dm 表"标记为"待确认（MCP 不可用）"，询问用户是否有已知的同主题 dm 表。

## 8.4.3 下沉判定

```
通用指标数 >= 3 且 无现有 dm 可扩列:
  → SINK-NEW: 建议新建 dm 宽表 + da 从 dm 取数
  → 升级为多表模式: task-1(dm新建), task-2(da), task-2.depends_on=[task-1]

通用指标数 >= 2 且 找到可扩列的 dm 表:
  → SINK-EXTEND: 建议扩列现有 dm + da 从 dm 取数
  → 升级为多表模式: task-1(dm扩列), task-2(da), task-2.depends_on=[task-1]

通用指标数 >= 2 且 无现有 dm 但指标计算简单:
  → SUGGEST: 提示可考虑下沉，展示分析明细
  → 用户确认下沉 → 同 SINK-NEW
  → 用户拒绝 → 保持单表

通用指标数 < 2 或 目标层=dm:
  → SKIP: 不建议下沉，保持单表
```
