# 多源消歧策略（Multi-Source Disambiguation）

当同一字段（如 `loan_amt`）在多张候选表中出现时，使用本策略选择最优数据源。

## 核心原则

**口径一致 > 粒度匹配 > 分层优先 > 覆盖率**

评分权重：

- 口径一致性：40
- 粒度匹配：30
- 分层优先：20
- 字段覆盖率：10

## Step 1：口径一致性（最高优先级）

先调用 `search_existing_indicators`：

- 命中且口径一致：直接采信 `source_table`，可跳过后续评分
- 命中但口径冲突：标记待确认
- 未命中：进入 Step 2

## Step 2：粒度匹配（30）

粒度来源优先级：

1. `TBLPROPERTIES.logical_primary_key`
2. 表注释中的 `[粒度:...]`
3. 分区键 + 非指标字段推断

粒度评分：

| 匹配情况 | 分数 | 说明 |
|---------|------|------|
| 完全匹配 | 30 | 表粒度 = 查询粒度 |
| 表粒度更细 | 15 | 需聚合 |
| 表粒度更粗 | 0 | 无法满足查询 |
| 无法判断 | 10 | 元数据缺失，保守给分 |

## Step 3：分层优先（20）

| 分层 | 分数 |
|------|------|
| `da_` / `ads_` | 20 |
| `dm_` / `dmm_` | 18 |
| `dws_` | 15 |
| `dim_` | 12 |
| `dwd_` | 8 |
| `ods_` | 2 |
| `tmp_` | 0 |

## Step 4：字段覆盖率（10）

```text
覆盖率分数 = (候选表命中目标字段数 / 总目标字段数) * 10
```

## Step 5：排序与确认

按总分降序输出 Top N：

- Top1 - Top2 >= 10：自动选择 Top1
- Top1 - Top2 < 10：触发用户确认

必须触发确认的场景：

- 指标口径冲突
- 分数接近（差值 < 10）
- 粒度无法判断
- 候选表跨业务域

## 对下游的输出契约

输出字段最少包含：

- `candidates`: 候选表及分数明细
- `selected_table`: 选中表
- `selection_reason`: 选择理由
- `need_user_confirm`: 是否需要确认

`generate-etl-sql` 在 Step 1 源表识别阶段消费该结果，并在 ETL 头注释保留“数据来源决策”。
